# Security Policy

## Supported Scope

Security reports are welcome for:

- `terminal-jarvis/packages/*`
- `terminal-jarvis/scripts/*`
- `terminal-jarvis/site/*`
- `.github/workflows/*`

## Reporting a Vulnerability

Please do not open public issues for sensitive vulnerabilities.

Report privately by emailing: `nnivetha373@gmail.com`

Include:

- Affected files/components
- Reproduction steps
- Impact assessment
- Suggested fix (if available)

## Response Targets

- Initial acknowledgment: within 72 hours
- Triage: within 7 days
- Fix timeline: depends on severity and complexity

## Security Defaults

Current project defaults include:

- Optional API key protection (`JARVIS_API_KEY`) for `/v1/*`
- Local-origin CORS defaults unless explicitly overridden
- Runtime warnings for risky configurations

## Disclosure

Please allow reasonable time for a fix before public disclosure.
