Getting Started

Set up in 3 steps

01

Install Ollama

Ollama runs LLMs locally. Install it first — it takes under a minute.

terminal 
# Linux / macOS
curl -fsSL https://ollama.com/install.sh | sh

# Windows — download from
# https://ollama.com/download/windows

After installing, verify with ollama --version and keep Ollama running in the background.

02

Model setup (automatic)

No manual baseline setup needed. The installer/launcher provisions chat, vision, and retrieval models on first run, then the desktop welcome page lets you choose any extra Ollama models from the live catalog.

terminal 
# Auto-pulled by the installer on first launch:
qwen2.5:3b    # preferred fast agent model
qwen2.5:1.5b  # fallback
qwen2.5       # fallback
qwen2.5vl:3b  # preferred live screen vision model
qwen2.5-vl:3b # fallback tag variant
llava:7b      # vision fallback
nomic-embed-text # local RAG embeddings
qwen2.5:3b — auto default qwen2.5:1.5b — fastest qwen2.5vl:3b — live screen qwen2.5 — fallback nomic-embed-text — RAG
03

Download & launch Jarvis

Grab the build for your OS, extract, and run the included launcher. No npm install is required.

terminal 
# Linux
tar -xzf terminal-jarvis-linux-x64.tar.gz
cd terminal-jarvis-linux-x64
./run-jarvis.sh

# Windows — extract zip, then run:
run-jarvis.bat

First run: click connect vault to link Obsidian, then click Google Sync once to import your schedule into local calendar context.

The first-run welcome screen also shows the live Ollama model catalog, excludes the included baseline set, and lets you pull extra models before entering chat.

Use Schedule On to include upcoming events in assistant context. Use + Event for local-only events and save reply to append answers into Jarvis/YYYY-MM-DD.md.

For Google import, set JARVIS_GOOGLE_CLIENT_ID once, restart the app, then click Google Sync.

Jarvis now auto-routes fast vs agent vs vision-capable models locally, remembers prior turns, and keeps an audit trail of context + tool activity at ~/.jarvis/audit/events.jsonl.
